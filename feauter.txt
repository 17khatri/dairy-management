# 🐄 Dairy Management System (Final Combined Spec for Codex)

## 📌 Project Overview

This is a **Dairy Management System** built using:

* Next.js (App Router)
* Tailwind CSS
* Prisma ORM
* PostgreSQL

The app is designed for a **single admin user** to:

* Manage suppliers (milk providers / cow boys)
* Record milk collection entries
* Track milk data (quantity, fat, price)
* Support **date-wise and session-wise (morning/evening)** entries
* Support **flexible pricing (default + override)**

## 🧱 Final Prisma Schema

```prisma
model Supplier {
  id           String   @id @default(uuid())
  name         String
  defaultPrice Float

  createdAt    DateTime @default(now())

  entries      MilkEntry[]
}

model MilkEntry {
  id         String   @id @default(uuid())

  supplier   Supplier @relation(fields: [supplierId], references: [id])
  supplierId String

  quantity   Float
  fat        Float

  // Optional override price
  price      Float?

  // Date-wise entry
  date       DateTime

  // Morning / Evening
  session    Session

  createdAt  DateTime @default(now())

  // Prevent duplicate entry for same supplier, date, session
  @@unique([supplierId, date, session])
}

enum Session {
  MORNING
  EVENING
}
```

## 🧠 Business Logic

### 💰 Price Handling

* Each supplier has a `defaultPrice`
* While creating entry:

  * If user provides price → use it
  * Else → use supplier defaultPrice

### 💡 Calculation

```ts
const appliedPrice = entry.price ?? supplier.defaultPrice
const total = entry.quantity * appliedPrice
```

## ⚡ API Requirements

### Suppliers API

#### GET `/api/suppliers`

* Return list of suppliers

#### POST `/api/suppliers`

```json
{
  "name": "Ramesh",
  "defaultPrice": 35
}
```

---

### Entries API

#### GET `/api/entries`

* Return all entries with supplier info

#### POST `/api/entries`

```json
{
  "supplierId": "abc123",
  "date": "2026-04-25",
  "session": "MORNING",
  "quantity": 10,
  "fat": 4.5,
  "price": 36
}
```

---

## 🚫 Constraints & Rules

* Only one entry allowed per:

  * supplier + date + session

* Do NOT update old entries if supplier price changes
